# StockWatch

## Описание проекта

StockWatch — это сервис мониторинга финансовых активов.

Пользователь может:

* отслеживать стоимость акций, ETF и криптовалют;
* создавать правила уведомлений;
* получать уведомления при выполнении условий;
* просматривать историю изменения цен;
* анализировать стоимость активов.

Проект разрабатывается как полноценный backend уровня production и предназначен для портфолио Python Backend Developer.

---

# Правила разработки

## API

В роутах FastAPI запрещено размещать бизнес-логику.

Роут должен:

* принять запрос;
* провалидировать данные;
* вызвать сервис;
* вернуть ответ.

Пример:

```python
@router.get("/{ticker}")
async def get_asset(
    ticker: str,
    service: AssetService = Depends(get_asset_service),
):
    return await service.get_asset(ticker)
```

---

## Service

В сервисах размещается вся бизнес-логика.

Именно сервис принимает решения:

* можно ли выполнить операцию;
* какие данные получить;
* какие репозитории вызвать;
* какие исключения выбросить.

Сервис ничего не знает о FastAPI.

---

## Repository

Репозиторий отвечает исключительно за работу с базой данных.

Он:

* выполняет SQLAlchemy-запросы;
* возвращает модели;
* не содержит бизнес-логики.

---

## Models

Все модели описываются через SQLAlchemy 2.0.

Использовать современный стиль.

Предпочтительно:

* Mapped
* mapped_column
* relationship

Не использовать старый стиль SQLAlchemy.

---

## Schemas

Для запросов и ответов использовать Pydantic.

ORM-модели никогда не должны возвращаться напрямую из API.

# Работа с базой данных

Использовать SQLAlchemy 2.0.

Предпочитать:

```python
stmt = select(Asset).where(Asset.ticker == ticker)
```

Не использовать:

```python
session.query(...)
```

Весь доступ к БД осуществляется через Repository.

---

# Dependency Injection

Использовать встроенный механизм Depends.

Не создавать сервисы вручную внутри роутов.

Правильно:

```python
Depends(get_asset_service)
```

---

# Получение рыночных данных

Источник данных должен быть абстрагирован.

Нельзя зависеть от конкретного API.

Использовать интерфейс.

Например:

```python
class MarketProvider(Protocol):

    async def get_price(
        self,
        ticker: str
    ) -> float:
        ...
```

Позже можно будет заменить API без изменения бизнес-логики.

Использовать JWT.

# Исключения

Создавать собственные исключения.

Например:

```python
AssetNotFoundError

AlertAlreadyExists

NotificationError
```

Не использовать HTTPException внутри сервисов.

HTTPException допускается только в API-слое или через глобальные обработчики исключений.

# Инструкции для Claude Code

При генерации кода необходимо:

* соблюдать существующую архитектуру проекта;
* не смешивать ответственность разных слоёв;
* писать production-ready код;
* использовать современные возможности Python 3.11+;
* использовать SQLAlchemy 2.0;
* использовать асинхронный код там, где это оправдано;
* добавлять type hints;
* писать понятный и читаемый код;
* не создавать огромные файлы;
* не писать бизнес-логику в FastAPI-роутах;
* при создании новой функциональности сразу предлагать тесты;
* если предлагается новая зависимость, кратко объяснять, зачем она нужна;
* при неоднозначностях выбирать решение, которое лучше подходит для долгосрочной поддержки и расширения проекта.

Главный принцип проекта — **чистая архитектура, расширяемость и код, максимально приближенный к коммерческой разработке**.
